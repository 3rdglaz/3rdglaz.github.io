Vou tentar fazer uma espécie de check-list que faz com que cada passo seja confirmado

Adicionando o Usuário e definindo Grupo

Antes de add temo que ter conhecimento a qual grupo pertence o diretório desejado /pasta/desejada/ → ls -ld /pasta/desejada . Devendo cuspir o seguinte:

drwxr-xr-x 14 $usuario $grupo 4096 Jul 30 02:24 /pasta/desejada
# guarde bem o GRUPO principalmente, já que é relativo a este grupo a 
# visibilidade da pasta ao final do acesso ao SFTP
# no caso eu só consegui com o grupo sendo ROOT em alguns casos

Caso não esteja assim é só setar 

sudo chmod 755 /pasta/desejada
# e dentro dela vc dá permissão a tudo com
sudo chmod 775 /pasta/desejada/*
# dessa forma o grupo tem full permissão ao conteudo, e visibilidade a esta past

Em seguida vamos adicionar o Usuario e setar o Grupo

sudo adduser $novoUsuario
# e add ao grupo
sudo usermod -a -G $grupoDaPasta $novoUsuario

SSH

Pra minha surpresa não tive que usar VSFTPD, usei apenas o SSH.

A probabilidade de vc ter que instalar isso é muito baixa

sudo apt install ssh
# e dentro do arquivo de configuração habilita acesso por senha

Configurando a Chave

Em uma máquina que não seja o server, vc cria um chave com ssh-key e sobe ela

ssh-copy-id -i caminhoda/chave/$nomeDaChavePrivada $usuario@$IpDoHost

Depois Desabilita o acesso por senha e verifica se o SSH com certificado tá rodando.

Configurando o SSH

Em seguimos configuramos o SSH pra permitir o compartilhamento específico ao usuário

sudo vim /etc/ssh/sshd_config

Lá procuramos por:

# vamos procurar e comentar
# Subsystem sftp /usr/lib/openssh/sftp-server
Subsystem sftp internal-sftp
# Utilizamos também uma nova métrica
Match User $nomeDoUsuario
				ChrootDirectory /pasta/desejada/
				ForceCommand internal-sftp
				X11Forwarding no
				AllowTcpForwarding no
				PasswordAuthentication yes

Salvamos e resetamos o serviço

sudo service sshd restart

Falar sobre mudanças caso haja ou não autenticação por senha ou certificado

Erros possíveis

Até o momento só encontrei o erro, seugindo esse processo, de Pipe fechada. Isso rola pq a /pasta/desejada/ não tá com a permissão correta no qual o usuário pertence ao grupo.
