Tutorial básico de como criar um site nas máquinas da DigitalOcean

Logando nas máquinas

Temos no total 3 VMs na digital Ocean

  1. VM guiando.com.br [cert]
guiando@198.211.117.8
  1. VM nexinvoice.com.br [cert]
guiando@164.90.217.163
  1. VM blog

dentro das VMs

Criamos uma pasta para o Subdominio

mkdir /var/www/$dominio.guiando.com.br
## recomendo criar um index.html para testes simples de conexão
vim /var/www/$dominio.guiando.com.br
# insere dentro
<p> dominio </p>
# salva né
:x

depois vamos Criar um arquivo no nginx

cd /etc/nginx/sites-avaible
# servidor Nexinvoice
cd /etc/apache2/sites-avaible

Podemos copiar algum .conf já existente e adequar ao novo $dominio

cp calculadorarpa.com.br.conf $dominio.guiando.com.br.conf

Agora precisamos trocar os índices de indicação

o arquivo copiado vai estar idêntico ao original:

server { server_name calculadorarpa.com.br www.calculadorarpa.com.br;

utilizando o VIM conseguimos trocar todas as strings necessárias

vim $dominio.guiando.com.br.conf
# agora em modo editor setamos
:%s/calculadorarpa/$dominio.guiando/g
# só salvar
:x

Agora apontamos o link referencial em sites-enabled

# no NEXINVOICE troca /nginx/ por /apache2/
cd /etc/nginx/sites-enable
# comando de link | seta -Source | usa esse apontamento
ln -s /etc/nginx/sites-available/$dominio.guiando.com.br.conf $dominio.guiando.com.br

Depois rodamos o CertBot pra confirmar tudo, caso seja necessário HTTPs

TODAVIA, é necessário [SUDO], e na VM só é permitido o uso de [SU -]

su -
# inserir senha 
certbot --nginx -d $dominio.guiando.com.br -d www.$dominio.guiando.com.br
# no NEXINVOICE trocar --nginx por --apache

SUBDOMINIO não precisa de www

Troca geral no VIM

# troca no arquivo/string 'foo'/ pela string'bar'/inclui tudo na mesma linha
:%s/foo/bar/g

Encaminhamento entre VMs

Criei um ambiente para site na VM Nex e propuseram um encaminhamento da para .guiando.com.br, por estarem em dominios diferentes tive que recorrer ao direcionamento da Amazon

  1. Abri o site da Amazon
  1. Procurei por S3
  1. Cliquei na opção Bucket
  1. Procurei pelo endereço que eu queria,
    storiescase.guiando.com.br
  1. Fui até o fim em Hospedagem de site estático
  1. Editei e selecionei
    1. Hospedagem de site estático
      ATIVADO
    1. Tipo de Hospedagem
      Solicitações de redirecionamento ...
    1. Nome do Host
      exemplo: stories.nexinvoice.com.br
    1. Protocolo
      nenhum
    1. Salvei
