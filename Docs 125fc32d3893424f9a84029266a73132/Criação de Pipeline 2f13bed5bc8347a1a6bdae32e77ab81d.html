Criando Pipeline

Em reunião com a equipe esse documento fora criadno em paralelo, podendo ter erros 15.50

Passos

1- Repositorio

2 olhar o projeto principal [.csproj]/ anotar

3 Ir no Pipeline → Pipeline

4 Pegar uma pipeline que já exista, e clonar,

Mudar o nome da Pipeline de acordo com Repositorio

Em get Sources apontamos com o nome do Repositorio

Em seguida, no JOB,

Em restore é padrão

Em dotnet build apontamos para o caminho do .csproj

Só salvamos e executamos

5 - a primeira rodada gera um artefato, sendo um destes uma DLL, que será publicada

Release de Publicação

Pega o artefato e publica no servidor.

Com o nome alteramos um CLONE,

e selecionamos o Artefato que geramos.

Em Task

Deploy é padrão

Deployment Group JOB

  • Deployment Group [ Recebe o Servidor que será publicado](normalmente padrão)
  • Artifact download[ apontamos o artefato]
  • PowerShell Script inicio Publicio
    • Webhook, devemos alterar o $username com o nome do crawler
  • Copy Files to ...
    • Pega a dll e copia na pasta do servidor
    • Source Folder = caminho do dll

Publicação Serviços Crawler - Roda a batch de publicação

PowerShell Script Sucesso - webhook de sucesso

PowerSHell Script Falha - webhook de falha

Sendgrid Email - Alerta por email

Em options, botamos o nome da release

Por ultimo, ativar CONTINUOUS DEPLOYMENT TRIGGER
