Mudar SSH Port

Qualquer IP público vai receber tentativa de acesso, principalmente se as portas forem padronizadas

  1. Backup

só criar um backup do (/etc/ssh/sshd_config)

cp /etc/ssh/sshd_config /etc/ssh/sshd_config.bkp
  1. Configurar

agora no arquivo de configuração principal

vi /etc/ssh/sshd_config

Procuramos pela linha de ports e alteramos pra porta que quisermos

# What ports, IPs and protocols we listen for 
Port 22
## depois da mudança vai ficar
Port 5675

Salvamos normalmente e saímos

  1. Restart

Reiniciamos o serviço para aplicar as configurações

service sshd restart
  1. Conectando
# protocolo | port 5675 | usuario@dominio
ssh -p 5675 usuario@ip

Força bruta

Muitas vezes haverão tentativas, com tecnologia antiga eu diria, de tentativa de conexão por força bruta, outras vezes por uma faixa de ips em cluster, outras vezes pela rede Thor utilizando bots.

Para o caso de IPs de cluster ou mesma rede, podemos utilizar o Fail2Ban. Dá update automático no iptables se atingir uma regra de limite.

  1. Instalando Fail2Ban
# pra ubuntu
apt-get install fail2ban

# CentOS
yum install epel-release
yum install epel fail2ban

Desativar autenticação por Senha

Autenticação por Chave vale mais a pena somente se o limite de usuário/dispositivos forem pequenos, 1 ou 2 computadores.

E caso seja de interesse segue

vi /etc/ssh/sshd_config

# Descomentar ou Adicionar o comando abaixo
PasswordAuthentication no

# resetar o serviço
service sshd restart

Fechar Portas

Mega básico, fechar as portas que não tem utilidade alguma, no caso de VMs para site

Port 80 e Port 443 podem se manter abertas, e obvio a porta de SSH.

Essa configuração vai rodar no serviços de Cloud
